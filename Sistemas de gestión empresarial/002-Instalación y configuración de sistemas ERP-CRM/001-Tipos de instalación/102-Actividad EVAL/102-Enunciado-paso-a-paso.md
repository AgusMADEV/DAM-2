# 102 - Actividad Evaluativa: Selección y Configuración de Plantillas en Sistemas ERP-CRM

## Objetivo
Evaluar la capacidad del estudiante para seleccionar, configurar y justificar el uso de plantillas apropiadas en sistemas ERP-CRM según las necesidades específicas de un proyecto empresarial.

## Contexto del Proyecto
Trabajas como consultor en sistemas ERP-CRM para una empresa mediana del sector comercial que necesita reorganizar la gestión de sus datos de clientes. La empresa maneja aproximadamente 500 clientes activos, realiza seguimiento de 50 oportunidades de venta mensuales y gestiona un catálogo de 200 productos.

## Paso 1: Entendiendo las plantillas disponibles

### 1.1 Análisis de Plantillas
Basándote en las plantillas disponibles en el archivo `001-Plantillas.md`, analiza cada una de las siguientes opciones:

**Plantillas Disponibles:**
- **Kanban**: Gestión visual de procesos y flujos de trabajo
- **Formulario**: Entrada y edición detallada de datos
- **Fichas de contacto**: Visualización de información de clientes y contactos
- **Calendario**: Gestión temporal de eventos y tareas
- **Lista/tabla**: Visualización estructurada de múltiples registros
- **Gráfica**: Representación visual de datos y métricas

### 1.2 Tarea a realizar
Crea una tabla comparativa que incluya:
- Tipo de plantilla
- Funcionalidad principal
- Ventajas
- Limitaciones
- Casos de uso recomendados

## Paso 2: Selección de la plantilla adecuada

### 2.1 Análisis de necesidades
Considera los siguientes escenarios empresariales y selecciona la plantilla más apropiada para cada uno:

**Escenario A: Gestión de pipeline de ventas**
- Seguimiento de 50 oportunidades mensuales
- Etapas: Prospecto → Contacto → Propuesta → Negociación → Cierre
- Actualización diaria por el equipo comercial

**Escenario B: Base de datos de clientes**
- Visualización de 500 clientes activos
- Información: datos de contacto, historial de compras, estado de cuenta
- Consultas frecuentes por múltiples departamentos

**Escenario C: Seguimiento de tareas comerciales**
- Programación de reuniones con clientes
- Fechas límite de propuestas
- Eventos de marketing y ferias comerciales

### 2.2 Justificación de selección
Para cada escenario, documenta:
- Plantilla seleccionada
- Factores considerados en la decisión
- Beneficios esperados
- Alternativas descartadas y por qué

## Paso 3: Configuración de la plantilla

### 3.1 Configuración detallada para el Escenario A
Selecciona la plantilla más apropiada para el pipeline de ventas y configura:

**Campos principales a incluir:**
- Datos del cliente potencial
- Valor estimado de la oportunidad
- Etapa actual del proceso
- Responsable comercial
- Fecha estimada de cierre
- Observaciones y notas

**Configuración de visualización:**
- Organización de la información
- Criterios de ordenación
- Filtros disponibles
- Indicadores visuales de estado

### 3.2 Optimización de eficiencia
Especifica las configuraciones que implementarías para:
- Facilitar la actualización rápida de datos
- Mejorar la visibilidad del estado de las oportunidades
- Permitir colaboración entre miembros del equipo
- Generar alertas automáticas para seguimiento

## Paso 4: Documentación del proceso

### 4.1 Manual de usuario
Crea una guía básica que incluya:
- Pasos para acceder a la plantilla configurada
- Instrucciones para actualizar información
- Interpretación de los indicadores visuales
- Procedimientos de mantenimiento básico

### 4.2 Beneficios implementados
Documenta cómo la configuración propuesta:
- Mejora la eficiencia operativa
- Facilita la toma de decisiones
- Reduce el tiempo de búsqueda de información
- Aumenta la colaboración entre equipos

## Restricciones y Consideraciones

### Limitaciones técnicas
- Solo utilizar las plantillas listadas en `001-Plantillas.md`
- No implementar funcionalidades externas al sistema base
- Mantener coherencia con los principios de instalación y configuración de sistemas ERP-CRM

### Criterios de evaluación
1. **Introducción y contextualización** (25%): Explica claramente cuáles son las plantillas disponibles y por qué es importante seleccionar la adecuada para el proyecto.
2. **Desarrollo técnico correcto y preciso** (25%): Describe cómo configuraste la plantilla seleccionada, incluyendo los campos y opciones utilizadas.
3. **Aplicación práctica con ejemplo claro** (25%): Proporciona un ejemplo práctico de cómo esta plantilla puede ser utilizada en el proyecto empresarial.
4. **Cierre/Conclusión enlazando con la unidad** (25%): Concluye explicando cómo la configuración correcta de las plantillas mejora la eficiencia del sistema ERP-CRM y cómo contribuye a lograr los objetivos del proyecto.

## Formato de entrega

### Estructura del documento
1. Portada con datos del estudiante
2. Tabla comparativa de plantillas (Paso 1)
3. Análisis y selección por escenarios (Paso 2)
4. Configuración detallada (Paso 3)
5. Manual de usuario (Paso 4)
6. Conclusiones y reflexiones

### Especificaciones técnicas
- Formato: Documento markdown (.md)
- Extensión: Entre 2000-3000 palabras
- Incluir diagramas o esquemas cuando sea necesario
